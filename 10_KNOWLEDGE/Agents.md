# AI Agents

## Simple Definition

An AI agent is an AI system that can pursue a goal through multiple steps, using tools, memory, planning, and feedback instead of only answering one prompt.

## Human Analogy

A chatbot is like a consultant giving advice. An agent is like an assistant that can take actions, check progress, and continue working.

## Core Components

- Model: the reasoning engine
- Instructions: goal, role, boundaries, and style
- Tools: APIs, browser, files, databases, code execution, enterprise systems
- Context and memory: what the agent knows about the current task and user
- Control loop: plan → act → observe → revise
- Guardrails: safety, approvals, permissions, fallbacks
- Evaluation: how we know whether it worked

## When Agents Are Useful

Agents are most useful when the task is multi-step, uncertain, tool-heavy, iterative, and valuable enough to justify automation.

Not every AI workflow needs an agent. Many useful systems are simpler workflows: classify, retrieve, summarize, draft, check, route, generate.

## Product Lens

The key product question is not: “Can we build an agent?”

The better question is:

> What workflow is painful enough, repeatable enough, and valuable enough to justify agentic automation?

## Enterprise Questions

- What should the agent be allowed to do without approval?
- What requires human confirmation?
- What tools/data should the agent access?
- How do we measure reliability?
- How do we prevent silent failure?
