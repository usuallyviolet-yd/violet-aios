# AI Memory

## Simple Definition

AI memory is the system that lets an AI assistant retain, retrieve, or use information across turns, sessions, or tasks.

## Human Analogy

Memory lets an AI assistant stop treating every conversation as the first conversation.

## Types of Memory

- Short-term context: what is in the current conversation window
- Saved memory: persistent user preferences or facts
- Episodic memory: previous events or interactions
- Semantic memory: stable knowledge extracted from past interactions
- Tool-based memory: external files, databases, GitHub notes, vector stores
- Agent memory: observations, plans, reflections, and task history

## Why Memory Matters

Memory improves personalization, continuity, and long-term usefulness. For agents, memory can support planning, reflection, and behavior over time.

## Risks

- storing wrong information
- over-personalization
- privacy concerns
- outdated assumptions
- unclear user control
- mixing temporary and permanent context

## Product Lens

Good AI memory is not just “remember everything.” Good memory means remembering the right information, with user control, auditability, and clear scope.

## Violet Connection

Violet AIOS is a deliberate external memory system. It should store durable, useful context that multiple LLMs can read, rather than relying only on one platform’s internal memory.
