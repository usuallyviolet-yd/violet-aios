# A2A — Agent-to-Agent Protocol

## Simple Definition

A2A is an open protocol that allows different AI agents to communicate, coordinate, and collaborate with one another.

## Human Analogy

If MCP helps an agent talk to tools, A2A helps one agent talk to another agent.

## MCP vs A2A

MCP:

- agent to tool
- agent to data source
- agent to workflow system

A2A:

- agent to agent
- cross-vendor collaboration
- multi-agent coordination

## Why It Matters

Enterprises may eventually have many specialized agents: sales, support, finance, design, compliance, project management, procurement, and operations. A2A aims to provide a shared communication pattern so agents can work across frameworks and vendors.

## Product Lens

A2A matters when a workflow crosses team, vendor, or system boundaries. For example, a design agent might need to coordinate with a cost-estimation agent, a compliance agent, and a scheduling agent.

## Open Questions

- How will agent identity and permissions work?
- How will agents verify each other’s trustworthiness?
- What happens when agents disagree?
- How should humans supervise multi-agent systems?

## Violet Connection

A2A is relevant to future enterprise AI and Spatial AI products where multiple specialized agents cooperate across design, construction, operations, and asset management workflows.
